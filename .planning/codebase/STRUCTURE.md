## Directory Purposes

**archetypes/**
- Purpose: Template for new post creation; defines default frontmatter
- Contains: Markdown files with YAML frontmatter templates
- Key files: `archetypes/default.md` - used by `hugo new` command

**content/**
- Purpose: All source content files; Hugo reads this directory during build
- Contains: Markdown files with YAML frontmatter (metadata + content)
- Key files:
  - `content/about.md` - About page (author bio, career history)
  - `content/post/*.md` - Individual blog posts (17 total posts about product management, behavioral science, UX)

**layouts/**
- Purpose: Override or extend theme templates; takes precedence over `themes/*/layouts/`
- Contains: Hugo template files (HTML with Go templating syntax)
- Key files:
  - `layouts/partials/head.html` - Custom HTML head block (meta tags, title, style/rss includes)
  - `layouts/partials/header.html` - Custom site header (logo, title, navigation, analytics)
  - `layouts/archetypes/default.md` - Custom post template (frontmatter + lorem ipsum skeleton)

**static/**
- Purpose: Static files copied as-is to build output; no processing
- Contains: Images and binary files
- Key files:
  - `static/image_test.jpg` - Test image file
  - `static/posts/{post-name}/` - Per-post image assets referenced in post content

**themes/hugo-notepadium/**
- Purpose: Hugo theme providing default templates, styling, and layout
- Contains: Theme files (Git submodule, not directly committed)
- Note: `config.toml` sets `theme = "hugo-notepadium"`; theme provides fallback templates if `layouts/` doesn't override

**config.toml**
- Purpose: Site configuration; controls Hugo behavior and theme parameters
- Contains:
  - `baseURL`, `title`, `theme` - Basic site settings
  - `languageCode`, `hasCJKLanguage` - Language configuration (Chinese configured, not used)
  - `googleAnalytics` - Analytics tracking ID
  - `[markup]` - Markdown rendering options (syntax highlighting, raw HTML)
  - `[params]` - Theme parameters (styling, logo, slogan, navigation)

**.planning/codebase/**
- Purpose: GSD codebase analysis documents
- Contains: ARCHITECTURE.md, STRUCTURE.md, CONVENTIONS.md, TESTING.md, CONCERNS.md
- Note: Generated by GSD `/gsd:map-codebase` command

## Naming Conventions

**Files:**
- Posts: `kebab-case.md` with leading uppercase or all lowercase
  - Examples: `hooked.md`, `DontMakeMeThink.md`, `sum_lean_product_playbook.md`, `Outcomes_over_outputs.md`
  - Pattern: No strict convention; mixed camelCase, snake_case, and PascalCase

**Directories:**
- Content sections: lowercase plural (`post/`)
- Static assets: lowercase or project-specific naming
- Theme directories: lowercase with hyphens (`hugo-notepadium`)

**Markdown Frontmatter:**
- Fields: lowercase with underscores or hyphens
  - `title`, `date`, `draft`, `description`
  - `tags`, `categories` - arrays
  - Pattern: Simple, flat structure; no nested fields

## Where to Add New Code

**New Blog Post:**
- Source file: Create `content/post/post-name.md`
- Template: Use `archetypes/default.md` via `hugo new post/post-name.md`
- Frontmatter: Include `title`, `date`, `draft`, `description`, `tags`, `categories`
- Assets: Add images to `static/posts/{post-name}/`
- Reference images: Use path `/posts/{post-name}/image.png` in Markdown

**New Page (non-post):**
- Source file: Create `content/pagename.md`
- Frontmatter: Add `title`, `date`, `draft` at minimum
- Assets: Place in `static/` or subdirectory as needed

**Custom Layout/Partial:**
- If overriding theme template: Create in `layouts/` with same path as theme
- Example: Theme file at `themes/hugo-notepadium/layouts/partials/nav.html` → override at `layouts/partials/nav.html`
- Pattern: Hugo checks `layouts/` first; falls back to theme if not found

**Custom Styling or JavaScript:**
- If theme supports partial hooks: Use `layouts/partials/` (e.g., `head-extra.html`, `footer-extra.html`)
- Otherwise: Modify `layouts/partials/head.html` or `layouts/partials/header.html`
- Alternative: Modify theme directly (not recommended; breaks updates)

**Navigation Items:**
- Edit `config.toml` → `[[params.nav.custom]]` section
- Add entries with `title` and `url` fields
- Theme renders from configuration
