Research findings:
- Astro 6 requires Node.js 22.12.0+ (odd-numbered versions unsupported)
- @astrojs/tailwind integration is deprecated for v4 — use @tailwindcss/vite instead
- Tailwind v4 uses CSS-native @theme directives instead of JavaScript config
- Zero-JS default makes Astro ideal for static portfolio content
</context>

<task type="auto">
  <name>Task 2: Install and configure Tailwind v4 via Vite plugin</name>
  <files>package.json, astro.config.mjs, src/styles/global.css</files>
  <action>
Install Tailwind v4 Vite plugin (NOT @astrojs/tailwind — that's deprecated for v4):

```bash
npm install -D @tailwindcss/vite
```

Update astro.config.mjs to use Tailwind's Vite plugin:

```javascript
import { defineConfig } from 'astro/config';
import tailwindcss from '@tailwindcss/vite';

export default defineConfig({
  vite: {
    plugins: [tailwindcss()],
  },
});
```

Create src/styles/global.css with Tailwind v4 CSS-native import and @theme scaffold:

```css
@import "tailwindcss";

@theme {
  /* Design tokens will be added in Plan 02 */
}
```

DO NOT create tailwind.config.js — Tailwind v4 uses CSS-native @theme directives instead of JavaScript config. Creating a config file is an outdated v3 pattern that causes conflicts.
  </action>
  <verify>
    <automated>grep -q "@tailwindcss/vite" package.json && grep -q "tailwindcss()" astro.config.mjs && grep -q "@import \"tailwindcss\"" src/styles/global.css</automated>
