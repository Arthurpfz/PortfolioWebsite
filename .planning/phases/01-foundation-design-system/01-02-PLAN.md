Extract and implement the design system from nousresearch.com as CSS custom properties organized in a three-layer token hierarchy, with test infrastructure for validation.

Purpose: Establishes the visual foundation for all components and pages. Without documented design tokens, implementation becomes ad-hoc with hardcoded values, leading to inconsistency and difficult theming. The three-layer approach (primitive → semantic → component) enables systematic styling and future theme variations. Test infrastructure enables verification of DSGN-05 typography requirement.

Output: Complete design token system documented in JSON, implemented as CSS variables in three layers, with test scaffolding for smoke tests, ready for component development in Phase 2.
</objective>

  <files>.planning/phases/01-foundation-design-system/design-tokens.json</files>
  <action>
Extract design tokens from nousresearch.com using browser DevTools and CSS Peeper extension.

**Typography extraction (DSGN-05):**
1. Open https://nousresearch.com in Chrome
2. Use CSS Peeper to quickly grab typography scale
3. For each heading level (H1-H6) and body text:
   - Record font-size (in px, will convert to rem)
   - Record line-height (unitless ratio preferred)
   - Record font-weight
   - Record letter-spacing if non-zero
   - Record font-family
4. Calculate scale ratio (H1 size / H2 size) to identify pattern (e.g., 1.2 = minor third scale)

**Color extraction (DSGN-01):**
1. Use DevTools Elements panel → Computed tab
2. Inspect background colors, text colors, button colors, border colors
3. Record hex/rgb values
4. Group by semantic role: Which colors are used for actions (buttons, links)? Backgrounds? Text? Borders?
5. Identify if colors follow a scale (e.g., gray-50, gray-100, gray-200... gray-900)

**Spacing extraction (DSGN-01):**
1. Inspect padding and margin values on containers, text blocks, buttons
2. Record all unique spacing values
3. Find mathematical relationship (greatest common divisor = base scale)
4. Document as scale: base, 2x, 3x, 4x, etc.

**Effects extraction (DSGN-01):**
1. Record box-shadow values (elevation system)
2. Record border-radius values
3. Record any transitions/animations durations

Document all extracted values in `.planning/phases/01-foundation-design-system/design-tokens.json` using W3C Design Tokens format:

```css
@layer primitives {
  :root {

    --font-size-base: 1rem;

    --font-size-2xl: 1.5rem;
    --font-size-3xl: 1.875rem;
    --font-size-4xl: 2.25rem;

    --line-height-tight: 1.25;
    --line-height-normal: 1.5;
    --line-height-relaxed: 1.75;

    /* Color primitives (extracted shades) */
    --color-blue-500: #...;
    --color-blue-600: #...;
    --color-gray-50: #...;
    --color-gray-900: #...;

    /* Spacing primitives (base scale) */
    --spacing-1: 0.25rem;
    --spacing-2: 0.5rem;
    --spacing-4: 1rem;
    --spacing-6: 1.5rem;
    --spacing-8: 2rem;

    /* Effects primitives */
    --radius-sm: 0.25rem;
    --radius-md: 0.5rem;
    --shadow-sm: 0 1px 2px 0 rgb(0 0 0 / 0.05);
  }
}
```

**src/styles/tokens/semantic.css** (Layer 2: Purpose-driven):
```css
@layer semantic {
  :root {
    /* Semantic typography */
    --text-h1: var(--font-size-4xl);
    --text-h1-line-height: var(--line-height-tight);
    --text-h2: var(--font-size-3xl);
    --text-h2-line-height: var(--line-height-tight);
    --text-body: var(--font-size-base);
    --text-body-line-height: var(--line-height-normal);

    /* Semantic colors */
    --color-background: var(--color-gray-50);
    --color-foreground: var(--color-gray-900);
    --color-action-primary: var(--color-blue-600);

    /* Semantic spacing */
    --space-content: var(--spacing-4);
    --space-section: var(--spacing-8);
  }
}
```

**src/styles/tokens/components.css** (Layer 3: Component-specific):
```css
@layer components {
  :root {
    /* Navigation */
    --nav-height: 4rem;
    --nav-padding: var(--spacing-4);

    /* Cards/containers */
    --card-padding: var(--spacing-6);
    --card-radius: var(--radius-md);
    --card-shadow: var(--shadow-sm);

    /* Buttons */
    --button-radius: var(--radius-md);
    --button-padding-x: var(--spacing-4);
    --button-padding-y: var(--spacing-2);
  }
}
```

Key principle: Components reference semantic tokens, semantic tokens reference primitives, primitives are raw values. Never skip layers (component directly to primitive).

```css
  --font-size-xl: 1.25rem;
  --font-size-2xl: 1.5rem;
  --font-size-3xl: 1.875rem;
  --font-size-4xl: 2.25rem;
}

/* Base typography styles using semantic tokens */
h1 {
  font-size: var(--text-h1);
  line-height: var(--text-h1-line-height);
  font-weight: 700;
}

h2 {

h3 {
```

CRITICAL: Values in @theme must match primitives.css exactly. @theme creates Tailwind utilities (bg-blue-500, text-gray-900), while :root in token files makes them available as CSS variables.

Note on duplication: Yes, primitive values appear twice (once in primitives.css :root, once in @theme). This is intentional — :root makes them CSS variables, @theme generates Tailwind utilities. Semantic/component tokens only go in :root (they don't need utilities).
