Deploy to Cloudflare Pages using Wrangler CLI (Cloudflare's official tool).

Install Wrangler:
```bash
npm install -D wrangler
```

Authenticate with Cloudflare (opens browser for login):
```bash
npx wrangler login
```

Create Cloudflare Pages project and deploy:
```bash
npx wrangler pages deploy dist --project-name=arthurpfz-portfolio
```

This creates a new Cloudflare Pages project named "arthurpfz-portfolio" and deploys the dist/ directory. Wrangler outputs the deployment URL (format: arthurpfz-portfolio.pages.dev).

For automatic deployments on future commits, connect Cloudflare Pages to GitHub repository via dashboard:
1. Visit https://dash.cloudflare.com → Workers & Pages → arthurpfz-portfolio
2. Click "Settings" → "Builds & deployments"
3. Connect to GitHub repository
4. Configure build settings: Build command `npm run build`, Build output directory `dist`

This enables automatic deployments whenever you push to main branch.

  <what-built>
Cloudflare Pages deployment complete. Now need to configure custom domain (arthurpfz.com) and verify SSL certificate.

Custom domain setup requires access to domain registrar to update DNS settings. Two options:

**Option A: Full Cloudflare zone (recommended)**
- Transfer nameservers to Cloudflare (requires registrar access)
- Cloudflare automatically creates CNAME record for Pages deployment
- Provides additional features (analytics, firewall, etc.)

**Option B: CNAME record only**
- Add CNAME record at current DNS provider pointing to arthurpfz-portfolio.pages.dev
- Simpler but fewer features

After DNS configuration, Cloudflare Pages Universal SSL will automatically issue certificate (takes 5-10 minutes).
  </what-built>
  <how-to-verify>
**Step 1: Configure custom domain**

Visit https://dash.cloudflare.com → Workers & Pages → arthurpfz-portfolio → Custom domains → "Set up a domain"

Enter: `arthurpfz.com`

**If using Option A (Full zone):**
1. Add arthurpfz.com as a Cloudflare zone (Sites → Add a site)
2. Copy Cloudflare nameservers shown in dashboard
3. Update nameservers at domain registrar to point to Cloudflare
4. Wait for DNS propagation (5-30 minutes)
5. Cloudflare auto-creates CNAME record pointing to Pages deployment

**If using Option B (CNAME only):**
1. In Cloudflare Pages custom domain setup, it will show required CNAME record
2. Add CNAME record at your current DNS provider:
   - Name: @ (or arthurpfz.com)
   - Target: arthurpfz-portfolio.pages.dev
3. Wait for DNS propagation (5-30 minutes)

**Step 2: Verify SSL certificate**

After DNS propagates, Cloudflare Pages automatically issues SSL certificate via Universal SSL.

Check certificate status:
```bash
curl -vI https://arthurpfz.com 2>&1 | grep -i "SSL certificate"
```

Should show valid certificate from Let's Encrypt or Cloudflare.

**Step 3: Test site accessibility**

1. Visit https://arthurpfz.com in browser
2. Verify placeholder homepage loads ("Foundation Ready")
3. Check browser address bar shows padlock icon (valid HTTPS)
4. Click padlock → verify certificate is valid and issued to arthurpfz.com

**Step 4: Verify DNS resolution**

```bash
dig arthurpfz.com
```

Should resolve to Cloudflare IP addresses or CNAME to *.pages.dev domain.

**Expected outcomes:**
- https://arthurpfz.com loads with valid SSL certificate
- Browser shows secure connection (padlock icon)
- Page displays "Foundation Ready" placeholder content
- DNS resolves to Cloudflare infrastructure
