  <name>Task 1: Enable Astro build optimizations</name>
  <files>astro.config.mjs</files>
  <action>
Add performance optimizations to Astro config:

1. Enable HTML compression: `compressHTML: true`
2. Enable inlineStylesheets: `inlineStylesheets: 'auto'` (inlines critical CSS)
3. Configure build output for production

Pattern:
```javascript
export default defineConfig({
  build: {
    inlineStylesheets: 'auto',
  },
  compressHTML: true,
  vite: {
    plugins: [tailwindcss()],
    build: {
      cssCodeSplit: true,
      minify: 'esbuild',
    },
  },
});
```

These settings reduce HTML size, inline critical CSS for faster first paint, and enable CSS code splitting to load only required styles per page.</action>

<task type="auto">
  <name>Task 2: Optimize article images with lazy loading</name>
  <files>src/pages/archive/[slug].astro</files>
  <action>
Update article markdown rendering to add lazy loading to images:

1. Find img tag styling section in [slug].astro prose styles
2. Add loading="lazy" attribute to all rendered images
3. Add width/height attributes if dimensions are known (prevents layout shift)
4. Add decoding="async" for non-blocking image decode

Use Astro's rehype plugins if needed, or apply via CSS-based approach. Since markdown is rendered via Content component, check if prose styles can add these via selectors. If not, add rehype-plugin to astro.config.mjs:

Option A (CSS - simpler):
```css
article img {
  max-width: 100%;
  height: auto;
  /* Note: loading and decoding must be in HTML, not CSS */
}
```

Option B (Rehype plugin - better):
Install `rehype-plugin-image-native-lazy-loading` or create custom rehype plugin to add attributes to img tags.

**Use Option A** if possible since site uses minimal dependencies. If images don't have loading="lazy" in rendered HTML, note in task completion that this is a limitation of current markdown rendering and suggest manual attribute addition to source markdown files.</action>
